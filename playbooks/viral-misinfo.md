# 谣言/断章取义扩散

## 场景描述
虚假信息或断章取义的内容在社交平台大规模传播。可能是误会、可能是恶意编造、也可能是旧信息被重新解读。典型：陈年旧事被翻出来、视频被剪辑成完全相反的意思、完全捏造的聊天记录/内部文件。

## 严重程度

| 级别 | 触发条件 |
|---|---|
| 🔵 苗头 | 单平台少量转发 |
| 🟡 发酵 | 多平台扩散，开始有人在评论区求证 |
| 🟠 爆发 | 被信任的KOL/大V转发背书，形成"实锤"认知 |
| 🔴 危机 | 主流媒体报道，或引发实际商业损失 |

## 应对步骤

### 1. 冷静期
- 不要每条谣言都追着辟。有些太离谱的，大众自己会分辨
- 判断是否值得回应：传播量、可信度、是否影响核心业务

### 2. 信息收集
- [ ] 追踪谣言源头：谁最先发的？什么时间？
- [ ] 判断传播动力：自然传播还是有组织推？
- [ ] 收集原始素材（完整视频、完整聊天记录、时间线证据）
- [ ] 统计各平台传播数据，判断是否已达回应阈值

### 3. 定性判断

| 情况 | 策略 |
|---|---|
| 传播量小、可信度低 | 不回应（回应反而给曝光） |
| 传播量大、有具体证据可反驳 | 正面辟谣 |
| 断章取义型 | 放出完整上下文 |
| 完全捏造型 | 辟谣 + 法律手段 |
| 旧事重提型 | 承认过去 + 说明现在 |

### 4. 回应策略

**核心原则：辟谣要快、要短、要带证据。一条谣言用三句话回应，不要写小作文。**

- 声明标题直接说结论："XX内容为虚假信息"
- 每条指控配一条证据，不要空口说"这是假的"
- 如果证据不方便公开（涉及隐私/商业机密），说明不方便公开但不代表默认
- 严重造谣直接律师函 + 报警，同时向平台投诉

### 5. 执行清单
- [ ] 确认谣言的核心 2-3 条指控（不要罗列十几条次要的）
- [ ] 准备证据材料（完整截图、原始文件、时间线）
- [ ] 撰写辟谣声明（用 denial 模板）
- [ ] 通过官方渠道发布
- [ ] 向主要传播节点（大V/媒体）私信说明真实情况
- [ ] 向平台举报不实信息
- [ ] 严重情况下发律师函/报警

## 回应节奏

```
谣言出现 2-4h 内：第一份辟谣声明
后续 24h：持续在各平台评论区附上辟谣链接
一周后：如果谣言已平息，不再主动提起
```

## 特别提醒

- 辟谣不要用"谣言止于智者"这种话 — 容易被理解成说信的人是傻子
- 不要因为急着辟谣而泄露不该泄露的信息（客户隐私、商业数据）
- 辟谣声明下面会有人继续杠，不要逐条回复
- 有些谣言会"辟谣反而更火"，提前做好心理准备
